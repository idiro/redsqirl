Synchronous Source

Description

Synchronous Source is an action that defines a synchronous data set. A synchronous dataset contains a timestamp within its path to identify its creation date. Typically a synchronous dataset does not change for a given period of time after its creation. Red Sqirl supports two data storage for synchronous data-set: HCatalog and HDFS.

Quick Guide

  • Inputs - No Inputs
  • DataType - Select the Data Storage (HDFS, HCatalog...)
  • SubType - Select the Data Format (Text Format, Compressed Format...)
  • Browser - Select a path, Change the Header
  • Template: define a template path
  • Occurrence: Define the frequency of the data
  • Outputs - The template path as well as the real path of the initial dataset

Data Type - Page 1

Data type defines the data storage used.

Data Sub-Type - Page 2

The data sub-type defines the format of your data. For example if the data storage is a file system the data could be file, a directory, compressed or not etc.

Browser - Page 3

Select Data

Browser is where the data source can be selected by browsing (depending on Data Type). The directory/table/file must be selected for the screen to progress

Browser Attributes

After the data source has been selected the data format can be viewed and modified. Column names can be renamed and the type of data the column will store can also be viewed

Template Path - Page 4

You can define here what are the parts of your path that define the timestamp. You will be able to use the following variables for that:

  • ${YEAR}
  • ${MONTH}
  • ${DAY}
  • ${HOUR}
  • ${MINUTE}

Path Selection - Page 5

In here you will be able to define the frequency of your dataset, and the one you need for your analyses. We list the three different parameters here:

  • Unit: The time unit of the frequency (year, month, day, hour or minute)
  • Frequency: The dataset refreshes every >n minutes
  • Number of instances: The number of the most recent instances to use for your analyses
  • Offset: The workflow runs at time t with a dataset timestamp labeled t+n*freq. A negative number means a delay between the data set and the running.
