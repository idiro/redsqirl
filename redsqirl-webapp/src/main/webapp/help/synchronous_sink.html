Synchronous Sink

Description

Synchronous Sink is an action that registers a synchronous data set at the end of a scheduled job. A synchronous dataset contains a timestamp within its path to identify its creation date. Typically a synchronous dataset does not change for a given period of time after its creation. Red Sqirl supports two data storage for synchronous data-set: HCatalog and HDFS.

In order to reuse the Sink dataset into another coordinator, you need to link it with a Synchronous Sink Filter

Quick Guide

  • Inputs - Input to register
  • Template path - The path in which the data is stored
  • Outputs - The registered output

Template Path - Page 1

The path with at least one of the following variable:

  • ${YEAR}
  • ${MONTH}
  • ${DAY}
  • ${HOUR}
  • ${MINUTE}
