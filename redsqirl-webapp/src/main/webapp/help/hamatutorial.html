Hama Tutorial

Hama Workflow

This workflow will demonstrate the Hama Logistic regression action. This action allows us to run logistic regression over a data set. It’s inputs are the training set and the prediction dataset. That means that two source actions are needed.

There are two files in the /tmp/ directory on keane called training.txt and the other prediction.txt. Transfer these files onto the hadoop file system into two new directories as performed in the previous tutorial. Use “,” as the delimiter in both source actions but use “id STRING , height FLOAT, colour String, age INT , churner INT” as the header for the “training,txt” file, and use “id STRING , height FLOAT, colour String, age INT” as the header for the “prediction.txt” file,

  • Create a new canvas by clicking the plus symbol on the canvas tabs bar
  • create the two source actions with the two files provided on keane
  • On the actions footer below click the actions property Icon
  • Either edit the current actions tab by click the title of it or create a new tab by inputting the new tab title in the field provided and click the “i” symbol
  • once in a configurable tab , click the checkbox beside “hama_logistic_regression” and then ok
  • make sure you are on tab that was just configures to include hama logistic regression and drag it to the canvas
  • select the source that is set to use the “training.txt” file and create a link to the new “hama LR” action.
  • a new window should appear and ask you to select whether it is a training data set or a prediction data set, select training and then click ok
  • select the source that is set to use the “prediction.txt” file and create a link to the “hama LR” action
  • in this window select prediction and then click ok
  • once the “hama LR” action has the two required inputs , open it and change the element id if required
  • the first page with list three interactions : Label , ID and Label Value.
  • in the Label interaction select churner
  • in the Id interaction select id
  • finally in the Label Value interaction input 1 as the value
  • click next to see the Model settings such as predictors and parameters for running the model.
  • For the purposes of the is tutorial we will leave these interactions alone and just click next and ok
  • Run the workflow

Summary of workflow

  • added actions to an actions tab
  • labeled the inputs for logistic regression
  • configured and ran a logistic regression
