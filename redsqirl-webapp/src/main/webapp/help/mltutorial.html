Machine Learning Tutorial

This workflow will demonstarte how to run a Logistic Regression in Red Sqirl. The Spark Logistic Regression from the SparkML package or the Hama Logistic Regression from the Hama package can be used.

You will need two files to complete this tutorial.

  • ml_tutorial_training_data.txt
  • ml_tutorial_prediction_data.txt
Transfer these files onto the hadoop file system into two new directories.

Hama/Spark Workflow

This workflow will demonstrate the Hama/Spark Logistic regression action. This action allows us to run logistic regression over a data set. It’s inputs are the training set and the prediction dataset. That means that two source actions are needed.

Use “,” as the delimiter and “ID STRING, SEPAL_LENGTH FLOAT, SEPAL_WIDTH FLOAT, PETAL_LENGTH FLOAT, PETAL_WIDTH FLOAT, SPECIES CATEGORY ” as the header in both source actions.

  • Create a new canvas by clicking the plus symbol on the canvas tabs bar
  • create the two source actions with the two files provided
  • On the actions footer below click the actions property Icon
  • Either edit the current actions tab by click the title of it or create a new tab by inputting the new tab title in the field provided and click the “i” symbol
  • once in a configurable tab , click the checkbox beside “hama_logistic_regression”/“spark_logistic_regression” and then ok
  • make sure you are on tab that was just configures to include hama/spark logistic regression and drag it to the canvas
  • select the source that is set to use the training file and create a link to the new “hama LR”/“spark LR” action.
  • a new window should appear and ask you to select whether it is a training data set or a prediction data set, select training and then click ok
  • select the source that is set to use the prediction file and create a link to the “hama LR”/“spark LR” action
  • in this window select prediction and then click ok
  • once the “hama LR”/“spark LR” action has the two required inputs , open it and change the element id if required
  • the first page with list three interactions : Label , ID and Label Value.
  • in the Label interaction select churner
  • in the Id interaction select ID
  • finally in the Label Value interaction input “Iris-setosa” as the value without quotes.
  • click next to see the Model settings such as predictors and parameters for running the model.
  • For the purposes of the tutorial we will leave these interactions alone and just click next and ok
  • Run the workflow

Summary of workflow

  • added actions to an actions tab
  • labeled the inputs for logistic regression
  • configured and ran a logistic regression
