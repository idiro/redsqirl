Hive Join

Description

Hive Join is an action allow you to join any number of table together, the result is a Hive table.A single type of join is performed through all the tables and the tables are joined one after the other in the order specified. The type of join supported are JOIN, LEFT OUTER JOIN,RIGHT OUTER JOIN and FULL OUTER JOIN.

Quick Guide

  • Inputs - at least two of either Hive Table or Partition
  • Features - Select the features from the inputs
  • Join Type - Select the Join type to be used in the join
  • Join Relationship - Select the relationship between the inputs
  • Output - Single Hive Table

Source

The source type for Hive Select has to be Hive and the sub types Hive Table or Hive Partition. Any number of inputs are allowed in Hive Join as long as there is at least one

Filters - Page 1

Filters will let the Join type be selected and add conditions for the join

Join Type

Join Type all types of join operations that can be performed

Conditions

Adding a condition to the join will filter out the data that do not meet the conditions applied

Join Relationship - Page 2

Join Relationship is where the relationship between the inputs is set. By selecting what columns from the inputs, the join relationship can be set

Join Operations

Join Operations provides the ability to copy the input data to the output data with the conditions and filters that have been set previously. Other operations can be performed on the data like casting and other data transformations.

IDM Hive

IDM Hive is the interface for IDM to interact with Hive on the hadoop cluster. There are two other actions that are apart of IDM Hive package Hive Select ,Hive Select and Hive Union.

Hive is a hadoop implementation of a relational database that is used for data warehousing for large data sets. Each query written in it's own language called HiveQl that acts like a normal SQL query but runs as a Map/Reduce job. Indexing is made easier and makes querying large datasets quicker and more efficient.
