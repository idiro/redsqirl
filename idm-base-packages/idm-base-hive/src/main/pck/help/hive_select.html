Hive Select

Decription

Hive Select is an action that is performed like any other Select statement in a relational database.

Quick Guide

  • Input - Single Hive Table or Partition
  • Features - Select the features to be used or calculated
  • Condition - Filter the selected data by a condition
  • Output - Set the output to support
    • Do Not Support Partition - Do not Create a partition
    • Support Partition in output - Create a partition and return the table
    • Support Partition in output and return only new partition - Create a partition and return the partition only
  • Output - Single Hive Table or Partition

Source

The source type for Hive Select has to be Hive and the sub types Hive Table or Hive Partition. Only one input is needed for Hive Select

Filters - Page 1

The Filters page will allow for organizing by conditions, partitions or grouping.

Condition

Condition is where the condition is set for the query. The attributes can be selected to be included in the query , from there the type of condition can be set based on the data type (String or Double) and what type of comparison or condition is set on the chosen attributes

Group By

In Group By the group by condition can be set to group the data set by an attribute

Feature Operations

Feature operations allows data to be copied to the output table and also do some transformation of the data. It allows casting of one data type to another type or other operations to the attribute.

IDM Hive

IDM Hive is the interface for IDM to interact with Hive on the hadoop cluster. There are two other actions that are apart of IDM Hive package Hive Join , Hive Aggregator and Hive Union.

Hive is a hadoop implementation of a relational database that is used for data warehousing for large data sets. Each query written in it's own language called HiveQl that acts like a normal SQL query but runs as a Map/Reduce job. Indexing is made easier and makes querying large datasets quicker and more efficient
