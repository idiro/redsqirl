Hive Aggregator

Quick Guide

  • Group By - Group the dataset by a selection of features
  • Features - Select the features to be used or calculated
  • Filter - Filter the selected data by a condition

Decription

Hive Aggregator is an action that is allows data to be aggregated .

Source

The source type for Hive Aggregator has to be Hive and the sub types Hive Table or Hive Partition. Only one input is needed for Hive Aggregator

Group By - Page 1

The Group By Interaction allows for features to be selected that the query will be grouped by.

Group By

In Group By the group by condition can be set to group the data set by an attribute

Feature Operations

Feature operations allows data to be copied to the output table and also do some transformation of the data. It allows casting of one data type to another type or other operations to the attribute.

There is a generator action that will generate the action depending on the generation type. If the generator action is select on MAX it will generate all the features available in the MAX function

IDM Hive

IDM Hive is the interface for IDM to interact with Hive on the hadoop cluster. There are two other actions that are apart of IDM Hive package Hive Select , Hive Join and Hive Union.

Hive is a hadoop implementation of a relational database that is used for data warehousing for large data sets. Each query written in it's own language called HiveQl that acts like a normal SQL query but runs as a Map/Reduce job. Indexing is made easier and makes querying large datasets quicker and more efficient
