Hive Union

Description

Union all input tables to display one output table that conatians all data defined in the union.

Quick Guide

  • Inputs - at least two of either Hive Table or Partition
  • Features - Select the features from the inputs
  • Join Type - Select the Join type to be used in the join
  • Join Relationship - Select the relationship between the inputs
  • Output - Single Hive Table

Source

The source type for Hive Select has to be Hive and the sub types Hive Table or Hive Partition. Any number of inputs are allowed in Hive Union

Operations - Page 1

Operations will allow for the input data to be copied to the output table if copy is selected, it also presents the table inputs and the attributes that are available to each table that are common. The data can be transformed or modified to suit the requirements of the workflow.

IDM Hive

IDM Hive is the interface for IDM to interact with Hive on the hadoop cluster. There are two other actions that are apart of IDM Hive package Hive Select , Hive Aggregator and Hive join.

Hive is a hadoop implementation of a relational database that is used for data warehousing for large data sets. Each query written in it's own language called HiveQl that acts like a normal SQL query but runs as a Map/Reduce job. Indexing is made easier and makes querying large datasets quicker and more eficient
