Pig Aggregator

Decription

Pig Aggregator is an action that allows data to be aggregated.

Quick Guide

  • Input
    • Text or Binary Map Reduce Directory
  • Configurations
    • Group By - Group the dataset by a selection of features
    • Transform data related to the group by
    • Sort data
    • Filter data row per row
    • Choose data format
    • Perform audit on the result
  • Output
    • Text or Binary Map Reduce Directory
    • audit (OPT): Control-A delimited directory with global audit values

Group By - Page 1

In Group By the group by condition can be set to group the data set by an attribute. When nothing is selected, the data set will be aggregated in one row.

Feature Operations - Page 2

Feature operations allows data to be copied to the output table and also to do some transformations of the data. It allows casting of one data type to another type or other operations to the attribute. Operations can be process over a list of rows defined by the group by clause.

Generator options are:

  • copy - copy the features that are grouped by
  • AVG - calculate the average on every aggregated number features
  • COUNT - count the number of non null values on every aggregated number features
  • COUNT_DISTINCT - count the number of distinct values on every aggregated number features
  • MAX - calculate the maximum value on every aggregated number features
  • MIN - calculate the minimum value on every aggregated number features
  • SUM - calculate the sum on every aggregated number features
  • AUDIT - same as generate AVG, COUNT, COUNT_DISTINCT, MAX, MIN and SUM

Ordering dataset - Page 2

In this page you can order the features at your convenience. If nothing is specified the data set will not be ordered.

Filtering, Formating and Auditing - Page 3

Condition

Conditions can be set to filter out the row of interest. The filter is applied row per row indepently. The query can be as complicated or as complex as needed.

Parallel

The parallel clause lets you choose the number of reducers used when ordering the dataset. This parameter is optional and useful if you have performance issues. Please read the performance guide, parallel section of the pig documentation.

Delimiter

Choose the delimiter character used to separate the output dataset field in each row. The delimiter is used only if you choose Text format.

Format

  • Text: save the result in a text format which is user readable, very handy for re-using the output
  • Binary: Use pig binary format. Binary format for reading and writing is a bit faster than a text format.

Audit

When the audit is enabled a second data set is generated with aggregate value that can give an overview to the user. For each column when relevant the range, the average, the number of non-null and null values as well as the number of distinct values (only for CATEGORY type) are calculated.

Red Sqirl Pig

Red Sqirl Pig is a package that contains actions that interface with Pig on the Red Sqirl platform to perform Pig actions. The pig actions available from the package are

  • Pig Aggregator
  • Pig Anonymise
  • Pig Binary Source
  • Pig Join
  • Pig Sample
  • Pig Select
  • Pig Text Source
  • Pig Transpose
  • Pig Unanonymise
  • Pig Union
  • Pig Value Binning
  • Pig Volume Binning

Redsqirl Pig Data set

The last page of every pig action will have two interactions that set the output format of the pig action. These interactions are to set the delimiter of the file for a TEXT MAP REDUCE DIRECTORY and the other is to set the output type of the file to be either TEXT MAP REDUCE DIRECTORY or BINARY MAP-REDUCE DIRECTORY

Auditing

After each Pig action it is possible to audit the result in order to have a better idea of the data set and help make decisions on what to do next. For example you may want to do an audit before using a binning node in order to have an idea of your value range. Also result of the audit may create auto-generated options.

Apache Pig

Apache Pig is a platform for analyzing large data sets, which consists of a high-level language for expressing data analysis programs, coupled with infrastructure for evaluating these programs. The salient property of Pig programs is that their structure is amenable to substantial parallelization, which in turns enables them to handle very large data sets. At the present time, Pig's infrastructure layer consists of a compiler that produces sequences of Map-Reduce programs, for which large-scale parallel implementations already exist (e.g., the Hadoop subproject). Pig's language layer currently consists of a textual language called Pig Latin, which has the following key properties:

  • Ease of programming. It is trivial to achieve parallel execution of simple, "embarrassingly parallel" data analysis tasks. Complex tasks comprised of multiple interrelated data transformations are explicitly encoded as data flow sequences, making them easy to write, understand, and maintain.
  • Optimization opportunities. The way in which tasks are encoded permits the system to optimize their execution automatically, allowing the user to focus on semantics rather than efficiency.
  • Extensibility. Users can create their own functions to do special-purpose processing.
