SNA Tutorial

SNA Workflow

This workflow will introduce the SNA package and how an SNA action can be compete. The data that is used is called “sna.txt” and is provided in the “/tmp/” folder on keane. Once this file has been copied onto hadoop’s fs , create a new workflow and add a source with this file. For the Header of the file use this configuration “A INT , B INT , Weight INT”. The actions for the SNA Package need to be added to the actions footer. The actions are page rank , line rank and effective closeness .

  • Once source is added and the actions are available in the actions footer drag a new page rank action to the canvas
  • create a link between the source and the new page rank action.
  • open the page rank action and change the id if required.
  • the first page contains three interactions which identify the three attributes for the algorithm.
  • for the “A” interaction select the “A” column from the drop down menu
  • for the “B” interaction select the “B” column from the drop down menu
  • the weight is optional in page rank but for this workflow select “Weight” in the drop down list and click next
  • the “Configurations” interaction allows for parameters to be modified for the algorithm , page rank has two : damping and workers.
  • For more information click the help link for page rank under the help tab for information about what each property.
  • click ok and then save and run the workflow

Summary of workflow

  • configured an SNA action
  • accessed the help menu
